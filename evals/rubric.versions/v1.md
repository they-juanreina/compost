# AI-suggestion eval rubric — v1

Versioned LLM-as-judge rubric for grading AI-authored events (codes, themes,
terms, annotations, insights). Emitted verdicts record this file's SHA so old
verdicts stay interpretable after the rubric is edited (a new file = new
version; never edit a shipped version in place).

Grade the suggestion on each dimension from 0.0 to 1.0, then give an overall
score (the minimum of the applicable dimensions — a suggestion is only as
trustworthy as its weakest link) and a one-paragraph explanation.

## Dimensions

1. **Faithfulness** (always) — Is every claim supported by the retrieved
   evidence? A claim with no supporting quote, or that contradicts the
   evidence, scores 0. Verbatim-quote support scores 1.

2. **Schema conformance** (always) — Does the suggestion include the required
   structured fields for its artifact kind (e.g. a code needs name +
   definition + ≥1 evidence anchor)? Missing required fields → 0.

3. **Novelty** (codes, themes) — Does this code/theme add signal not already
   covered by an existing code/theme? A near-duplicate scores low.

4. **Specificity** (terms) — Is the glossary term defined precisely enough to
   be useful, and anchored to a real usage in the corpus? Vague or generic
   definitions score low.

## Output

```json
{
  "verdict": "pass" | "fail",
  "score": 0.0,
  "dimensions": { "faithfulness": 0.0, "schema": 0.0, "novelty": 0.0, "specificity": 0.0 },
  "explanation": "…"
}
```

`verdict` is `pass` when `score >= 0.7`, else `fail`. Omit dimensions that do
not apply to the artifact kind.
